# Security Policy

## Supported Scope

Security reports are accepted for:

- authentication and session handling
- authorization and access control
- payment and webhook verification
- data exposure, injection, and file upload risks

## Reporting a Vulnerability

Do not open public issues for security vulnerabilities.

Report privately to the owner listed in [`LICENSE.md`](LICENSE.md) with:

- vulnerability summary
- impact and affected endpoints/files
- reproduction steps and proof-of-concept
- suggested fix (if available)

## Response Process

1. Acknowledge report receipt.
2. Validate and classify severity.
3. Prepare and test remediation.
4. Coordinate disclosure timeline with reporter if needed.

## Safe Harbor

Good-faith research is appreciated, but must:

- avoid privacy violations and service disruption
- avoid destructive testing
- remain within legal boundaries and license terms
